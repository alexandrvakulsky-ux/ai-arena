**Claude:** Found it. The root cause is straightforward:

**`SESSION_TOKEN` is regenerated every server restart.** After your Railway deploy, the server got a new token. Your browser had the old token in `sessionStorage`. When `init()` runs on page load and `/api/verify` returns 401, the code just does nothing — no `else` branch, no redirect to password prompt. So if you had the tab open, the app stays "unlocked" (UI visible) but every API call fails with 401 because `getToken()` returns the stale token.

The fix: add a 401 handler to `init()` that re-locks the app, so the user is prompted to log in again.
