## What Shipped

### Backend (server.js)
- ScrapeCreators API integration with smart fallback detection
- Detection triggers: policy-removed ads (>50%), zero returns, all inactive, stale results
- Fixed bug: 0-result pages weren't triggering fallback (was `totalCount === 0 && ads.length === 0` → `false`)
- Unix timestamp date fix for ScrapeCreators data
- Server-side pagination (`?page=N&limit=50`) — 25KB per page vs 3.4MB
- On-demand image proxy — fetches fbcdn images on first request, caches to disk
- Batch image download after scraping
- Puppeteer: 3 concurrent tabs, 1.5s delay, processes ALL active ads
- Discovery endpoint: manual search + auto-scan (30 keywords, 12h cache)
- Persistent watchlist (JSON on disk)
- Relevance filter + blocklist for discovery results
- Video proxy only flags `has_video` when actual URL exists

### Frontend (public/index.html)
- Intersection Observer with 400px rootMargin for lazy image loading
- Sentinel-based infinite scroll + JSON prefetch for page N+1
- Default view: active ads only
- Long text collapse (>200 chars, click to expand)
- Video play button with error handling
- Discover tab: auto-scan on open, manual search, keyword chips, Track/Remove buttons
- Ad Library links on discover results

### Code Cleanup
- Removed dead `groupVariants()` function
- Deduplicated discovery endpoints (shared helpers)
- Removed sync `fs.existsSync()` from health endpoint
- Removed unused `preview_cached` from normalizers
- Added README.md, .env.example
